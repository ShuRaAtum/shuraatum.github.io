# CV Site Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a bilingual (EN/KO) personal CV site for `ShuRaAtum`, based on the al-folio Jekyll theme, deployed via GitHub Actions to `https://shuraatum.github.io`.

**Architecture:** Fork/import the al-folio template into the existing repo, then layer bilingual mirroring (`/en`, `/ko`) on top via duplicated `_pages/` files, shared `_data/strings_{en,ko}.yml`, and a language switcher include. Content seeds (one BibTeX entry, one project, placeholder About + CV) prove the build green before real content is added. Deploy uses al-folio's bundled GitHub Actions workflow.

**Tech Stack:** Jekyll 4.x · Ruby 3.x · Node 18+ (for BibTeX rendering) · al-folio v0.x · Bundler · Pretendard (CDN) · GitHub Actions · GitHub Pages

**Spec:** `docs/superpowers/specs/2026-05-26-cv-site-design.md`

### Task 6: Create the language switcher include

**Files:**
- Create: `_includes/lang-switcher.html`

- [ ] **Step 1: Write the include**

`_includes/lang-switcher.html`:

```liquid
{%- comment -%}
  Swaps the first path segment between /en and /ko.
  Falls back to /en/ if neither prefix is present.
{%- endcomment -%}
{%- assign current_lang = page.lang | default: "en" -%}
{%- if current_lang == "en" -%}
  {%- assign other_lang = "ko" -%}
{%- else -%}
  {%- assign other_lang = "en" -%}
{%- endif -%}
{%- assign current_path = page.url -%}
{%- assign other_path = current_path | replace_first: "/" | split: "/" -%}
{%- assign rest = "" -%}
{%- for seg in other_path offset:1 -%}
  {%- assign rest = rest | append: "/" | append: seg -%}
{%- endfor -%}
{%- assign target = "/" | append: other_lang | append: rest -%}
{%- if target == "/en" or target == "/ko" -%}{%- assign target = target | append: "/" -%}{%- endif -%}

{%- assign current_key = "strings_" | append: current_lang -%}
{%- assign current_strings = site.data[current_key] -%}
<a class="lang-switcher" href="{{ target | relative_url }}" hreflang="{{ other_lang }}">
  {{ current_strings.lang.switch_to_other }}
</a>
```

- [ ] **Step 2: Commit**


## Notes for the implementer

- al-folio's file names and layout structure change between releases. If a file referenced in this plan (`_layouts/bib.html`, `_includes/header.html`, etc.) is missing or differs, find the equivalent by grepping for the closest analogous string (e.g. `entry.pdf` for the bib button). Update the task in place rather than blocking.
- Run `./scripts/smoke.sh` after every task. If it fails, fix before moving on.
- Commit frequently — every task above ends in a commit on purpose. If a task is interrupted, the repo state is always recoverable.
- Don't pre-fill real content (papers, projects, photo, bio) in this phase. The plan's job is to make the site build green with the right shape; real content goes in afterward as ordinary edits.
- For the placeholder PDFs, the `printf` trick produces a technically-valid 1-page PDF. Some PDF viewers may complain — it's only for the deploy smoke test. Replace ASAP.
